A · 並排預覽

  • 所見即所得 + 原始碼並存
  • 最易上手
  • 元件 → 即時看
  • 資訊較少（無 TOC/Frontmatter）
最適合：混合寫 md + Svelte 標籤的人

B · WYSIWYG + Palette

  • 閱讀體驗最像成品
  • 拖曳插入元件很順手
  • Frontmatter 表單化
  • 編 raw 標籤需切模式
最適合：Obsidian 使用者、純寫作者

C · 三欄全配

  • 功能最完整
  • TOC + Frontmatter + Palette 全部一次有
  • 適合長文
  • 小螢幕會擠
最適合：重度部落格作者

D · 專注 WYSIWYG

  • 排版最美、留白大
  • 右欄可收合
  • ⌘K 指令面板為主
  • 進階功能需學習快捷鍵
最適合：寫作為主、偶而嵌元件

下一步 · 挑一個做高保真

我的推薦是 C（三欄全配） —— 因為你列了很多功能（多分頁、搜尋、Frontmatter、TOC、Git）而且想要元件 palette 拖曳。C 可以一次把它們擺得最舒服；寫作時右欄也可以收合變成類似 D 的專注模式。告訴我要哪一個，我就做高保真。
